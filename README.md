# ⌨ SETUP

**사기 전에** 스펙이 아니라 **감각**으로 데스크 셋업을 발견하고, 그 취향이 **카드가 되어 밖으로 퍼지는** 서비스.
커머스 이전 단계(취향 발견) 구현. → 전략: [`../setup-strategy-v5.md`](../setup-strategy-v5.md)

## 실행
```bash
npm install
npm run dev       # http://localhost:5173
npm run build     # 프로덕션 번들
npm run typecheck # 타입 체크
```

## 구현된 코어 루프 (v5 §2)
```
[진단] 이미지·소리만 60초 → 취향 시드
  → [피드] 소리 나는 무드보드 + ♡ → 취향 게이지(시그니처)
  → [꾸미기] 역방향 추천(취향·소리·용도·예산 → 셋업) + 픽터치(호환성 실시간)
  → [결과지] 아키타입 카드 (라이트/다크) → 공유(borrowed crowd)
```

## v5 디자인 원칙 반영 (§4)
- **다크 퍼스트** — 그래파이트 `#101216` 기본, 사진이 발광하는 무대 (§4.2)
- **반커머스 문법** — 장바구니·할인·별점·좋아요 카운트 없음. 무드가 1차 내비 (§4.1)
- **취향 게이지 시그니처** — ♡ 누를 때 3축(미니멀↔맥시멀·웜↔쿨·조용↔화려)이 실시간 이동 (§4.2)
- **소리 나는 핀터레스트** — 무드 카드 누르면 타건음 재생 (§2.1)
- **결과지 = 공유 카드** — 아키타입 뱃지 + FAQ 3줄(불안 해소) (§4.5)
- **모바일 퍼스트** — 세로 화면이 원본, 하단 탭 내비 (§4.7)

## 구조
```
src/
  types.ts            도메인 타입 (Axes 3축 등)
  data.ts             제품·무드·아키타입 시드(큐레이션)
  engine.ts           역방향 추천 + 호환성 + 아키타입/감성어
  store.ts            취향·좋아요·빌드 상태(localStorage 영속)
  sound.ts            타건음 WebAudio 합성
  router.ts           해시 라우터
  components/         Keys(절차적 렌더) · TasteGauge · Nav
  screens/            Diagnose · Feed · Builder · Result
```
